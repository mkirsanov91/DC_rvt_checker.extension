# ТЗ: Скрипт проверки отверстий в инженерных системах
**Проект:** NED DC — DC_rvt_checker.extension  
**Вкладка:** NED → панель Checker → кнопка "Проверка отверстий"  
**Язык кода:** IronPython 2.7 (pyRevit)  
**Комментарии:** на русском  
**Параметры и значения статусов:** на английском  

---

## Контекст запуска
Скрипт запускается из **пустой рабочей модели NED DC**, в которую все остальные модели подключены как Revit Links:
- Модели конструктива: АР (архитектурная) и/или КР (конструктивная)
- Сводная модель отверстий (опционально)
- Одна или несколько моделей инженерных систем (MEP)

---

## Шаг 1 — Выбор моделей (UI диалог)

### 1.1 Выбор конструктивных моделей
Чекбоксы со списком всех подключённых Revit Links.  
Пользователь выбирает одну или несколько: АР, КР, сводная модель отверстий.

### 1.2 Выбор инженерных моделей
Чекбоксы со списком оставшихся Revit Links.  
Пользователь выбирает одну или несколько MEP моделей.

### 1.3 Настройки проверки
- **Зазор (мм):** числовое поле, по умолчанию 50 мм  
  *(минимальный зазор между трубой/воздуховодом и краем отверстия)*
- **Директория для Excel:** путь к папке сохранения, кнопка "Обзор..."  
  *(запоминается между запусками через pyRevit config)*

## Шаг 2 — Логика проверки

### 2.1 Определение пересечений
Для каждого MEP элемента (трубы, воздуховоды, лотки кабельные, короба):
- Получить Bounding Box элемента
- Проверить пересечение с геометрией стен и перекрытий из конструктивных моделей
- Если геометрия MEP элемента входит внутрь геометрии стены/перекрытия → **пересечение найдено**

### 2.2 Классификация стен и перекрытий
**Бетонные стены** — определяются по наличию слова **בטון** (бетон) в названии материала или типа стены.  
Остальные — архитектурные/некритичные.

**Цветовая градация по толщине** (применяется в Excel и навигаторе):
| Толщина | Цвет | Приоритет |
|---|---|---|
| ≥ 400 мм | 🔴 Красный | Критично |
| 200–399 мм | 🟠 Оранжевый | Важно |
| < 200 мм | 🟡 Жёлтый | Информация |

### 2.3 Проверка отверстий
Для каждого найденного пересечения:

**А) Физическое отверстие (вырез в геометрии):**
- Проверить есть ли вырез/проём в теле стены или перекрытия в месте пересечения
- Если MEP элемент проходит внутри внешних границ геометрии конструктива → считается что отверстие **прорезано**

**Б) Проверка размера отверстия:**
- Габарит отверстия должен быть больше габарита MEP элемента + зазор (заданный пользователем) с каждой стороны
- Если меньше → статус **"Undersized"**

**В) Пустое отверстие:**
- Отверстие есть в конструктиве, но ни один MEP элемент через него не проходит
- Статус **"Empty"**
- По мере проверки разных инженерных систем пустые отверстия заполняются данными
- После проверки всех систем оставшиеся пустые = **"Free"** (требуют заделки или использования)

### 2.4 Итоговые статусы пересечений
| Статус | Описание |
|---|---|
| `No Opening` | Пересечение есть, отверстия нет |
| `Opening OK` | Отверстие есть, размер достаточный |
| `Undersized` | Отверстие есть, но мало |
| `Empty` | Отверстие есть, MEP не проходит |
| `Free` | Отверстие пустое после всех проверок |

## Шаг 3 — Реестр отверстий в модели

### 3.1 Семейство отверстия
В рабочей модели NED DC размещаются семейства-маркеры для контроля.  
Каждое семейство содержит параметры:

| Параметр | Тип | Описание |
|---|---|---|
| `NED_Status` | Text | Статус: `Pending` / `Approved` / `Rejected` / `Free` |
| `NED_ApprovalDate` | Text | Дата изменения статуса (YYYY-MM-DD) |
| `NED_Comment` | Text | Комментарий проверяющего |
| `NED_OpeningSize` | Text | Габарит: L x H (мм) |
| `NED_ElevationFromLevel` | Number | Высота от уровня до низа отверстия (мм) |
| `NED_WallType` | Text | `Concrete` / `Architectural` |
| `NED_WallThickness` | Number | Толщина стены/перекрытия (мм) |
| `NED_MEP_ElementID` | Text | ID MEP элемента |
| `NED_Structure_ElementID` | Text | ID стены или перекрытия |
| `NED_System` | Text | Название инженерной системы/модели |
| `NED_Level` | Text | Этаж/уровень |

### 3.2 Логика согласования
- Пользователь вручную меняет `NED_Status` в Revit
- При изменении на `Approved` или `Rejected` автоматически проставляется `NED_ApprovalDate`
- Статус `Pending` — ещё не рассмотрено

## Технические ограничения и допущения
- Работает только с элементами из **Revit Links** (не из основной модели)
- Пересечения определяются через **BoundingBox Intersection** (не точная геометрия — для скорости)
- Для точной проверки размера отверстия используется геометрия элемента
- IronPython 2.7 — использовать только совместимые библиотеки
- Excel генерируется через `openpyxl` (входит в pyRevit)
